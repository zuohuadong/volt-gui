# Architecture Design

## Purpose

Help teams choose architecture that fits constraints, preserves operability, and avoids unnecessary complexity.

## Workflow

1. Identify requirements, constraints, traffic, data sensitivity, and team capabilities.
2. Map components, ownership, data flow, and failure modes.
3. Compare options with explicit tradeoffs.
4. Define migration path, rollout plan, and observability.
5. Document decisions and assumptions.

## Output

Return:

- Recommended architecture.
- Component responsibilities.
- Tradeoff table.
- Failure modes.
- Migration and rollout plan.
