# Frontend UX Review

## Purpose

Help product teams improve web interfaces for real user workflows rather than visual polish alone.

## Workflow

1. Identify target users, key tasks, and frequency of use.
2. Review navigation, information density, hierarchy, forms, states, and errors.
3. Check accessibility, keyboard behavior, responsive layout, and text overflow.
4. Prioritize issues that block task completion or repeated use.
5. Provide concise design and implementation recommendations.

## Output

Return:

- Findings ordered by impact.
- Affected screen or component.
- Recommended change.
- Verification steps.
