# Test Plan And QA

## Purpose

Turn requirements and changes into a practical verification plan with clear risk coverage.

## Workflow

1. Identify user journeys, changed behavior, dependencies, and failure modes.
2. Split tests into unit, integration, E2E, performance, security, compatibility, and manual checks.
3. Prioritize high-risk paths and production-critical regressions.
4. Define pass/fail criteria and required evidence.
5. Produce a concise execution checklist.

## Output

Return:

- Scope and assumptions.
- Test matrix.
- Regression checklist.
- Required test data.
- Release gate criteria.
